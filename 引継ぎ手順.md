## 4. ファイル一式の入手方法

### 方法A：GitHubから ZIP ダウンロード（最も簡単）

1. https://github.com/kanno-beep/okuderamedical-rentalstudio にアクセス
2. 緑色の **「Code」** ボタンをクリック
3. メニューから **「Download ZIP」** を選択
4. ZIPを解凍 → そのままFTPでサーバーへアップロード

### 方法B：GitHub と git で連携（更新が継続する場合に推奨）

GitHub のリポジトリを clone して使う：

```bash
git clone https://github.com/kanno-beep/okuderamedical-rentalstudio.git
cd okuderamedical-rentalstudio
```

今後の修正があれば `git pull` で最新版を取得できます。

> ※ private に切り替える場合や、コラボレーター追加（kanno-beep様にご相談）が必要なら別途ご連絡ください。

## 5. サーバーへのアップロード手順（FTPの場合）

### 公開URLの計画

御社の公開URL構造に合わせて配置を選んでください：

| 公開URL案 | サーバー上の配置 |
|-----------|-----------------|
| `https://example.com/` （ドキュメントルート） | サーバーの`public_html/` 直下 |
| `https://example.com/studio/` （サブディレクトリ） | `public_html/studio/` 配下 |
| `https://studio.example.com/` （サブドメイン） | サブドメイン用ディレクトリ |

> **電光板QRコードとの整合性**：最終的な公開URLが決まったら、QRコードの差し替えが必要です（後述）。

### FTPアップロードの注意点

1. **ASCII 転送ではなく BINARY 転送** を使用してください（画像が壊れる原因）
2. **日本語ファイル名のフォルダ `Webイラスト/`** はそのまま転送
   - FTPクライアントが UTF-8 対応であることを確認
   - WinSCP / FileZilla は UTF-8 既定でOK
3. **大文字小文字を保持**（特に Linux サーバー）

### 動作確認

アップロード後、以下を確認：

- [ ] トップページが開く（`/index.html`）
- [ ] 4枚のヒーロー画像が自動で切り替わる
- [ ] プログラムカード16枚すべての画像が表示される
- [ ] ハンバーガーメニューが開閉する（スマホ）
- [ ] 予約フォームのステップが正しく動く

## 6. QRコードの更新（電光板用）

サイトを正式URLに公開した後、A1電光板のQRコードの差し替えが必要です。

### 自動生成版（既存）

サイト内 `index.html` の `#reservation` セクション末尾に、**そのページのURLを指す QRコードを自動生成** するJSが入っています（`js/main.js` 内 `initQRCode`）。

サイト本体を新URLにアップロードすれば、自動で新URL用のQRコードが表示されます。

### 印刷物用 QRコード

電光板に印刷するQRコードは別途生成が必要です：

1. 正式公開URLが確定（例：`https://okuderamedical.co.jp/studio/`）
2. 無料QR生成サービス（[QR Code Generator](https://www.qr-code-generator.com/) 等）で生成
3. 高解像度 PNG/SVG でダウンロード（A1サイズに耐える解像度）
4. 既存の電光板デザインに差し替え（A1: 84.1×59.4cm想定）

## 7. 予約フォームの動作について

✅ **本実装済み**（サーバー側 PHP で送信）

外部サービス（FormSubmit.co 等）には**一切依存しません**。
サーバー内の `contact.php` が PHP の `mb_send_mail()` で直接メールを送信します。

- **受信先メールアドレス**: `okudera@carrot.ocn.ne.jp`
- **自動返信**: お問い合わせ者がメールを入力していた場合、自動で確認メールを送信
- **件名（受信）**: 「【貸しスタジオ】Webサイトから新しいお問い合わせがありました」
- **件名（自動返信）**: 「【貸しスタジオ】お問い合わせ受付のご連絡（オクデラメディカル）」
- **送信情報**: お名前・電話・メール・利用目的・利用時間・希望日時・ご要望
                受信日時・送信元IPもログ目的で含まれます

### サーバー要件

- PHP **7.0 以降**
- `mail()` 関数 または `mb_send_mail()` が利用可能
- 上記が使えるレンタルサーバーであれば追加設定不要

（さくらインターネット、Xserver、ロリポップ、お名前.com 等の主要レンタルサーバーは
標準で対応しています）

### 配置先

`contact.php` を `index.html` と**同じディレクトリ**にアップロードしてください。

```
public_html/             ← 公開ディレクトリ
├── index.html
├── contact.php         ← ★ 必須
├── css/
├── js/
└── Webイラスト/
```

### ⚙️ 初回設定（重要）

`contact.php` 冒頭の「設定」セクションを必ず編集してください：

```php
// 受信先メールアドレス（オクデラ様のメール）
$SHOP_EMAIL = 'okudera@carrot.ocn.ne.jp';

// 送信元アドレス
// ★ サーバードメイン上に存在するアドレスを設定してください
// 例： noreply@okuderamedical.co.jp
$FROM_EMAIL = 'okudera@carrot.ocn.ne.jp';
$FROM_NAME  = 'オクデラメディカル 貸しスタジオ';
```

⚠️ **`$FROM_EMAIL` は注意点があります**

多くのレンタルサーバーでは「From にサーバードメイン以外のアドレス（例：他社の
@carrot.ocn.ne.jp）を指定するとスパム判定される・拒否される」という制約があります。

公開ドメインが決まったら（例： `studio.okuderamedical.co.jp`）、
そのドメイン上で**実在するメールアカウント**を作成して From に指定するのが
最も安定します。例：

- `$FROM_EMAIL = 'noreply@okuderamedical.co.jp';`
- もしくは現状の `$SHOP_EMAIL` と同じ値（同一ドメイン送信）

`Reply-To` ヘッダーには自動的にお問い合わせ者のメールが入るので、店舗側で
返信ボタンを押すとそのまま相手にメールが戻ります。

### 動作確認

サーバーへのアップロード後：

1. ブラウザでサイトを開く（**HTTPS推奨**）
2. 予約フォームを最後まで進める
3. 「📩 この内容で送信」をクリック
4. 「お問い合わせありがとうございます！」の画面が出れば送信完了
5. `okudera@carrot.ocn.ne.jp` の受信箱で確認
6. テストで自分のメールを入れた場合、そちらに自動返信が届くか確認

### トラブルシューティング

| 症状 | 原因と対処 |
|------|-----------|
| 「サーバー応答の解析に失敗しました」エラー | `contact.php` がPHPとして実行されていない。<br>サーバーのPHP有効化、ファイルパーミッション（644）を確認 |
| メールが届かない | `$FROM_EMAIL` をサーバードメインのアドレスに変更<br>迷惑メールフォルダ確認<br>サーバー会社にメール送信制限の有無を確認 |
| 文字化け | サーバーが UTF-8 出力に設定されているか確認 |
| 自動返信だけ届かない | お問い合わせ者のメールサーバー側でブロック中の可能性 |

### 受信先メールアドレスを変更する場合

`contact.php` 冒頭の `$SHOP_EMAIL` の値を編集するだけです。

### 自動返信文を変更する場合

`contact.php` の `$bodyAuto = ...` の本文を編集してください。

## 9. ブラウザキャッシュ対策（重要）

CSS/JS を更新したのに表示が変わらない場合、ブラウザキャッシュが原因です。

### 対策：CSS/JS にバージョン番号を付ける

`index.html` の以下の箇所を、更新のたびに `?v=日付` の値を変えてください：

```html
<link rel="stylesheet" href="css/style.css?v=20260528">
...
<script src="js/main.js?v=20260528"></script>
```

これで訪問者のブラウザが必ず最新版を取得します。
